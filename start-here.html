How to Edit and Deploy Your Digital vCard

This guide explains how to personalize your vCard using the provided vcard folder template. You'll edit the content, update meta tags, and upload everything to your hosting platform.

1. Folder Structure

The folder you received is named vcard. All paths inside the code are already configured to work with this folder name, so you do not need to rename it.

2. Files Overview

  • index.html - Page structure and metadata
  • vcard.bundle.js - Main configuration file (update cardData)
  • styles/style.min.css - Styling and layout
  • scripts/qrcode.min.js - QR code generation
  • images/ - Profile, cover, and sharing images (must stay named the same unless paths are updated)
  • assets/ - PSD templates to help recreate images in the correct size and format (PNG)
Important: If you replace image files in images/, use the same filenames and dimensions (e.g. headshoot-300x300.png). If you use different names, you must update the paths in both index.html and vcard.bundle.js.

3. What to Edit

In vcard.bundle.js:

Locate the cardData object. Customize your information such as name, title, contact info, bio, company details, social links, and image filenames (inside images/).

In index.html:

Edit these specific lines to personalize your metadata, favicon, and social sharing:


<meta name="author" content="John Smith">
<meta name="description" content="The digital business card of John Smith - ready to save, share, and connect.">
<title>John Smith Digital Business Card</title>
<meta name="robots" content="noindex, nofollow"> <!-- Optional: remove this if you want it indexed -->

<link rel="icon" href="https://yourdomain.com/vcard/images/favi-100x100.png">
<link rel="apple-touch-icon" sizes="128x128" href="https://yourdomain.com/vcard/images/bookmark-128x128.png">

<meta property="og:type" content="website">
<meta property="og:title" content="John Smith Digital Business Card">
<meta property="og:description" content="The digital business card of John Smith - ready to save, share, and connect.">

<meta property="og:image:width" content="1200">
<meta property="og:image:height" content="630">
<meta property="og:url" content="https://yourdomain.com/vcard/">

<meta name="twitter:card" content="summary_large_image">
<meta name="twitter:title" content="John Smith Digital Business Card">
<meta name="twitter:description" content="The digital business card of John Smith - ready to save, share, and connect.">
<meta name="twitter:image" content="https://yourdomain.com/vcard/images/share-1200x630.png">

3.5 Preview and Customize Locally (Recommended)

Before going live, it's a good idea to test and customize your digital vCard in a safe local environment. We recommend downloading and installing LocalWP - a free and beginner-friendly app that lets you run a local WordPress server on your computer.

Even though this project doesn't require WordPress to work, LocalWP is perfect for viewing and editing static HTML files in a controlled browser environment.

Steps:

  1. Download and install LocalWP from https://localwp.com
  2. Create a new "Custom" site (you can skip WordPress installation options)
  3. Open the site folder created by LocalWP on your computer
  4. Copy your vcard folder into \app\public directory
  5. Enable SSL
  6. In your browser navigate to https://yourwebsite/vcard/index.html
Why use LocalWP?
It ensures that links, scripts, and QR code previews work correctly in a browser environment, before you upload to your live server. It's also safer and faster than uploading with every small edit.

4. Deploy Your vCard

  1. Replace image files in the images/ folder (optional, must match dimensions and names).
  2. Edit vcard.bundle.js to update the content.
  3. Edit index.html to update meta tags and image URLs.
  4. Upload the entire vcard folder to your host public_html directory (e.g. WordPress/cPanel/Netlify).
  5. Visit your URL: https://yourdomain.com/vcard/index.html

5. Enable SEO (Optional)

This tag tells search engines not to index this page or follow its links. Important: It is only a *request*, not a guarantee. Some bots may still index the page despite this.

If you want your vCard to appear in search engine results (like Google), you'll need to remove or comment out the following line in index.html:

<meta name="robots" content="noindex, nofollow">

Styling Guide for Digital vCard

This guide explains how the styling system works across your digital business card. It uses a modular, variable-driven CSS structure for easy customization. Each section below matches your actual class names and describes their function and design behavior.

1. Global & Base Styling

  • :root - Declares all CSS variables used across the project (colors, font sizes, spacing, etc.).
  • html - Applies the main background color via --main-bg-color.
  • body - Centers the card layout and sets base font color.
  • #body - Applies the base sans-serif font family.
  • .main-style - White card background via --card-bg-color.

2. Modal (QR Code Overlay)

  • #qr-modal - Modal container with show/hide transitions.
  • .modal-hidden, .modal-open, .modal-closed - Visibility states toggled via JavaScript.
  • #qr-modal_header - Flex row holding the close button.
  • #qr-modal-close - Clickable close icon button.
  • .modal-title and .modal-description - Centered modal texts styled using modal color variables.
  • #qr-code - QR box with background and rounded corners.

3. Header & Action Icons

  • header - Wraps the header area and allows icon positioning.
  • .header-cover - Grid layout for responsive cover image.
  • #header-cover-image - Full-width banner image with top-center crop.
  • #header-actions - Sticky top-right icon bar.
  • .icon-svg, .icon-svg-share, .icon-svg-qr, .icon-svg-close - Standard icons using stroke.
  • .icon-svg-whatsapp - Filled WhatsApp icon with scale effect.

4. Profile Section

  • #profile - Flex column layout for image + info.
  • #profile img - Circular profile photo with fixed size.
  • #profile-info - Holds name, title, and address.
  • .profile-name - Large bold name text.
  • .profile-job-title - Job role text just below the name.
  • .profile-bio - Paragraph for personal/brand tagline.
  • .profile-address - Faded address styling with small font.

5. Contact Buttons

  • .contact-actions - Main flex layout for buttons.
  • .button-wrapper, .button-wrapper-inner - Per-button layout and spacing.
  • .button-call, .button-email, .button-whatsapp - Circular action buttons with background color.
  • .button-icon-wrapper - Sizes icons inside each button.
  • .button-text-color - Sets white text using --save-contact-text.
  • #save-contact, .button-save-contact - Pill-style button for saving contact.

6. Company Info & Reviews

  • .company-description - Descriptive company text block.
  • .company-info-background - Light gray box using --company-box-bg.
  • .company-info-content - Rounded container for company section.
  • .company-info-stars, .star-full, .star-half, .star-empty - Star coloring for reviews.
  • .company-info-reviews - Text like ‚Äúbased on X reviews‚Äù.

7. Footer

  • footer - Overall footer layout and spacing.
  • .site-footer - Applies light background via --footer-bg-color.

8. Utility & Helpers

  • .text - Center-aligned text block used in footers or general sections.
  • .close-btn-invert - Inverts close icon color if needed.
  • .symbol-defs - Hides SVG symbol block (used with <use>).
  • #vcf-save-contact - Optional alternate save contact section.
