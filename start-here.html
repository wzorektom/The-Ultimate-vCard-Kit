1. Folder Structure

The folder you received is named vcard. All file paths are configured to work with this folder name. Do not rename it unless you also update all references in index.html and vcard.bundle.js.

2. Files Overview

  • index.html - Page structure and metadata for browsers and social media
  • vcard.bundle.js - Configuration object where all your profile info is stored
  • styles/style.css - Complete styling using modular CSS variables
  • scripts/qrcode.min.js - QR code generator (do not modify)
  • images/ - Contains your profile photo, banner, and social preview images
  • assets/ - Optional PSD templates to recreate your images to correct size
Important: If you change image filenames, you must update the paths in both vcard.bundle.js and index.html. Make sure to save .png images in the same dimensions as provided in the name of the file.

3. What to Edit

In vcard.bundle.js:

Open this file and locate the cardData object at the top. Every visual element (name, phone, links, etc.) is configured here. You only need to change the text inside quotation marks. Do not remove keys or curly brackets.

In index.html:

In index.html: Update your SEO and social preview metadata inside the <head> tag. Be sure to update your domain, your full name (first and last), and make sure all paths to preview images are correct.

These image paths are used by platforms like WhatsApp, Facebook, Twitter (X), LinkedIn, and Messenger when someone shares your digital business card link. If the image URLs are properly set and point to live, publicly accessible files, these platforms will automatically pull the correct preview image when generating link cards.

Tip: You can test how your link will look when shared using tools like:
  • Facebook Sharing Debugger
  • LinkedIn Post Inspector
  • Twitter (X): Simply paste the link into a tweet and post it - Twitter will automatically generate a preview card if your meta tags are correct.
If your image doesn't show up as expected, double-check the full path and clear the platform's cache using these tools (or repost the link).

If you plan to use this vCard for multiple users on the same domain (e.g. team members, staff, or clients), it's a good idea to place each user's vCard in their own folder.

4. Deploy Your vCard

  1. Replace image files in the images/ folder making sure the dimensions are the same
  2. Update vcard.bundle.js with your real data
  3. Edit index.html to update metadata and preview images
  4. Upload the entire vcard folder to your public_html or root folder on any other hosting
  5. Access it at: https://yourdomain.com/vcard/index.html
Tip: Changes Not Showing Up?
If you've uploaded your vCard to a live server and made edits (like replacing images, updating text, or modifying code) but don't see the changes, it's likely due to caching.

‚úÖ You should clear the following caches:
  • Browser cache: Hard refresh (Ctrl+F5 or Cmd+Shift+R), or clear cached files manually
  • WordPress cache plugins: e.g. WP Rocket, W3 Total Cache, LiteSpeed Cache
  • Cloud services: If you're using Cloudflare, Cloudways, or similar - log in and purge the cache
  • Server-level cache: Some hosts (like SiteGround or Kinsta) offer built-in caching that also needs to be cleared

Changes won't reliably show up for you (or your visitors) until all layers of caching are cleared.

5. Enable SEO (Optional)

Your site includes this line by default:

<meta name=\"robots\" content=\"noindex, nofollow\">

This tells Google and other search engines not to index the page. Remove it (or comment it out) if you want your page to be listed publicly.

Important: The <meta name="robots" content="noindex, nofollow"> tag is only a request to search engines. While most major crawlers like Google, Bing, and Yahoo will respect it, some bots (especially aggressive or third-party ones) may still index or archive your page anyway.

‚ö†Ô∏è If you want stronger protection, consider adding password protection at the server level or using proper HTTP authentication. However, for general SEO control, removing or commenting out this meta tag is usually enough to allow indexing by Google and other mainstream search engines.

Social Profiles

Your social media links are configured inside the cardData object in vcard.bundle.js. Each profile is defined with a type and a url in the socialProfiles array. You can easily add, remove, or comment out profiles based on what you want to display.

Here's what the full block looks like by default:

numberOfIcons: 3, // Set how many social icons you want to show - 3 by default

socialProfiles: [
  { type: "youtube", url: "https://youtube.com/yourlogin" },
  { type: "tiktok", url: "https://tiktok.com/yourlogin" },
  { type: "twitter", url: "https://twitter.com/yourlogin" },
  /*
  { type: "facebook", url: "https://facebook.com/yourlogin" },
  { type: "instagram", url: "https://instagram.com/yourlogin" },

  { type: "nextdoor", url: "https://nextdoor.com/yourlogin" }

],

To use a profile, simply uncomment the line by removing the // or removing the /* ... */ block if it surrounds multiple lines. For example, to enable LinkedIn: 

{ type: "linkedin", url: "https://linkedin.com/yourname" },

How to Control How Many Icons Are Displayed

The numberOfIcons property determines how many of your social profiles will appear visually on the card. By default, it's set to 3 - meaning only the first 3 profiles in the list will be shown, regardless of how many are defined.

Important: You must make sure that the number of uncommented profiles matches the value of numberOfIcons. For example, if you only enable two profiles, set:

numberOfIcons: 2

If you enable five profiles but leave numberOfIcons at 3, only the first three will be displayed. The others will be ignored in the visible layout but still exist in the data structure (e.g. for the downloadable .vcf file).

Review Settings

This section controls the star rating display and the link to your public Google Business Profile (or review page). These values are configured inside the cardData object in vcard.bundle.js. 


googleBusinessProfile: "https://g.page/yourbiz",
ratingValue: "4.8",
reviewLinkText: "Check Our Reviews on Google",

How It Works

  • googleBusinessProfile ‚Äî The full URL to your Google Business page or public review site. This will be used as the target when a user clicks the review link on your vCard.
  • ratingValue ‚Äî A numeric value from 0.0 to 5.0 (decimals required). This value is used to dynamically calculate how many stars should be shown.

    For example:
    • 4.0 shows four solid stars
    • 4.5 shows four full stars and one half-opacity star
    • 3.3 results in three solid stars and two faint ones
    ‚ö†Ô∏è Do not enter 4 (integer) ‚Äî always use a decimal format like 4.0, or the stars may not display correctly.
  • reviewLinkText ‚Äî This is the clickable text that appears next to your rating (e.g. ‚ÄúSee Our Reviews‚Äù, "Check Our Reviews on Google", ‚ÄúCheck Google Feedback‚Äù, etc.).

The stars are rendered automatically based on your ratingValue using a JavaScript function and styled with CSS opacity classes (from .opacity-0 to .opacity-100). No additional icons or markup are required.

Sharing API

These fields control the content used by the native mobile sharing feature ‚Äî activated when a visitor taps the ‚ÄúShare‚Äù button on your vCard. This functionality relies on the browser's built-in navigator.share() API, which is available in most modern mobile browsers (Chrome, Safari, Edge, and some Android apps). 


shareTitle: "Jane Doe Digital Business Card",
shareText: "You can view my Digital Business Card here:",

How It Works

  • When a user taps the share icon (three connected dots), the site checks if navigator.share() is supported. If it is, it opens the native share sheet of the device (iOS or Android), allowing the user to share the link via apps like Messages, Mail, WhatsApp, or Facebook.
  • If the browser doesn't support navigator.share() (e.g. older browsers or desktop), the fallback is to display the QR code modal instead, allowing the user to scan or screenshot the card link.

What These Fields Control

  • shareTitle ‚Äî The title shown in the mobile share dialog. Usually this is your name or a short description of the card (e.g. ‚ÄúJane Doe Digital Business Card‚Äù).
  • shareText ‚Äî The descriptive text shown under the title in the share prompt. This is the message someone will see before clicking the link ‚Äî keep it friendly and concise.

Styling & CSS Customization

Color Customization

The color system in this vCard has been designed to be easy to adjust using CSS variables. You can find all the main colors at the top of style.css under the :root selector.

These variables allow you to customise the branding, text, background, buttons, and modal styles in one place ‚Äî without editing individual styles manually.

Main Color Variables

:root {
  --color-primary: #0a40ab;        /* Main branding color - header, buttons, footer */
  --color-light-bg: #ffffff;       /* Light background - card content area */
  --color-dark-text: #000000;      /* Main text color */
  --color-light-text: #ffffff;     /* Light text - icons, buttons, footer */
}

Additional Theme Variables

  • --footer-bg-color - Footer background color
  • --company-box-bg - Company info section background
  • --modal-bg-color - Background for QR code modal
  • --star-color-full - Color of rating stars (default: gold)
  • --qr-code-color - Color of the generated QR code SVG
Example:
To create a minimal theme: 

--color-primary: #111827;
--company-box-bg: #f3f4f6;

Button & Icon Styling

Each action button (Call, Email, WhatsApp, Save Contact) is styled with its own class and controlled by dedicated variables:

  • .button-call ‚Üí --contact-button-bg
  • .button-email ‚Üí --contact-button-bg
  • .button-whatsapp ‚Üí --contact-button-bg
  • .button-save-contact ‚Üí --save-contact-bg and --save-contact-text

The icons inside these buttons use fill: currentColor to inherit color from the parent container. This makes them easy to theme.

Social Media Icon Styling

Social profile icons are styled per platform using CSS classes. Each platform gets its default brand color: 


.social-links a.linkedin   { background-color: #0077b5; }
.social-links a.facebook   { background-color: #0866FF; }
.social-links a.youtube    { background-color: #FF0000; }

You can override any of these classes to match your own brand theme. If you prefer a neutral design, set them all to a shared color like #333.

Tip: All social icons use fill: currentColor, so you can globally control their icon color with the root variable --social-icon-color.

Typography & Spacing

Font sizes and layout spacing are controlled using CSS variables, making the card layout easily scalable: 


--font-size-lg: 1.25rem;   /* Name and major headings */

--spacing-md: 1rem;

These variables ensure consistent spacing and sizing across all components. You can easily adjust them to make the layout feel more compact or more spacious.
