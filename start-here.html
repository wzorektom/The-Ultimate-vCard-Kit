How to Edit and Deploy Your Digital vCard

This guide explains how to customise your vCard using the provided vcard folder. Your vCard is JSON-driven, which means your contact content (name, phone, links, images, share text, etc.) is stored in one place: vcard/cardData.json.

Main workflow:
1) Edit cardData.json for content
2) Edit index.html only for social share preview + absolute URLs (Open Graph / Twitter / icons)

2. Files Overview

  • demo-1/ and demo-2/ - two complete example setups (same template, different styling + sample data)
  • vcard/cardData.json - main file you edit (text, links, image paths, share settings, etc.)
  • vcard/index.html - page shell + social sharing meta tags (title/description, Open Graph, Twitter/X, favicon/icon absolute URLs)
  • vcard/vcard.bundle.js - reads cardData.json and renders the page + vCard download logic
  • vcard/styles/style.css - styling + colour/theme variables
  • vcard/scripts/qrcode.min.js - QR code library (do not edit)
  • vcard/images/ - profile image, cover image, favicon, share preview images (PNG files used by the live vCard)
  • Assets/ - editable design source files (PSD templates) + matching exported PNGs (use these to update share-1200x630.png, favicons, cover/profile images, etc.)
  • start-here.html - optional helper / demo landing page (if included in your repo)
  • LICENSE.txt - open-source licence for this repository
Tip: If you edit a PSD in Assets/, export a new PNG with the same filename into vcard/images/ (or update the filename in cardData.json and the share/meta image URLs in index.html).
Important: If you change image filenames, update the matching paths in cardData.json (and in index.html for social share images/icons). Keep the same image dimensions where possible.

3. Edit Your Content in cardData.json

Open vcard/cardData.json. This file is the single source of truth for your vCard content. Everything you see on the page (and everything saved into the downloadable vCard) is pulled from here. You can safely edit the text inside quotes, but don't delete commas, brackets, or field names (keys), otherwise the JSON will break and the card won't load.

Tip: If the card suddenly stops loading after an edit, it's usually a JSON formatting issue (missing comma, extra comma, or missing quotes). Undo your last change, save again, and re-check.

Example Structure

Here's a real example (shortened) so you can see how values are stored: 

{
  "fullName": "Mira Solensen",
  "jobTitle": "Founder & Domestic Services Consultant",
  "company": "Purespace Property Care Ltd",
  "email": "mirasolensen@mycompany.com",
  "telNumber": "+440000000001",
  "websiteUrl": "https://mycompany.com",
  "address": "London, United Kingdom",
  "bio": "Purespace Property Care Ltd provides professional cleaning...",
  "profileImage": "images/headshoot-300x300.png",
  "coverImage": "images/cover-960x640.png"
}

Common Fields (What Each One Does)

FieldWhat it controlsExample value
fullName Your full name shown on the card and used in the vCard download. Mira Solensen
jobTitle The subtitle under your name (keep it short so it fits nicely). Founder & Domestic Services Consultant
company Your business/brand name (shown on the card and used in contact details). Purespace Property Care Ltd
birthday Your birthday stored in yyyymmdd format for the vCard file.
Example: 1st January 1990 = 19900101 		19900101
email Email used by the Email button and saved into the vCard. mirasolensen@mycompany.com
telNumber Main phone number used by the Call button and saved into the vCard.
Use international format if possible (recommended): +44... 		+440000000001
officeNumber Optional secondary number (office/landline). If you don't want it, keep it as an empty string or remove usage from your UI (depending on your template behaviour). +440000000002
whatsappNumber Number used by the WhatsApp button. Usually the same as telNumber.
Keep it in international format (+44...). 		+440000000001
websiteUrl Your website link (used for the website button/link and often included in sharing). https://mycompany.com
address Address shown on the card and stored in the vCard. This can be full address or just a city/area. London, United Kingdom
bio Short “about” sentence shown near the top of the card. Aim for 1-2 lines for best layout. Purespace Property Care Ltd provides professional cleaning...
companyDescription Longer description shown in the company/info box (you can be more detailed here). We specialise in deep cleaning, end-of-tenancy preparations...

Images

The image fields reference files inside vcard/images/. If you replace images, either: keep the same filename, or update these paths to match. 

{
  "profileImage": "images/headshoot-300x300.png",
  "coverImage": "images/cover-960x640.png"
}
Important: If you change image filenames used for social sharing (like share-1200x630.png), you must also update the absolute URLs inside index.html (Open Graph + Twitter tags), otherwise WhatsApp/Facebook/LinkedIn previews will still show the old image or none at all.

Footer and Sharing Text

These control your footer line and the text used by the mobile Share button: 

{
  "footerTagline": "Your Trusted Partner in Home and Property Care.",
  "footerCompanyUrl": "https://mycompany.com/",
  "footerCompanyName": "Purespace Property Care Ltd",
  "shareTitle": "Mira Solensen Digital Business Card",
  "shareText": "You can view my Digital Business Card here:"
}

Social Profiles

Social links are configured in the socialProfiles array. Each item supports: type, url, and enabled.

Show / hide a profile: set "enabled": true or false.
If enabled is missing, it's treated as enabled by default. 
"socialProfiles": [
  { "type": "youtube", "url": "https://youtube.com/yourlogin", "enabled": true },
  { "type": "tiktok", "url": "https://tiktok.com/@yourlogin", "enabled": false },
  { "type": "linkedin", "url": "https://linkedin.com/in/yourlogin" }
]

Reviews (Star Rating + Link)

{
  "googleBusinessProfile": "https://g.page/yourbiz",
  "ratingValue": "4.5",
  "reviewLinkText": "Check Our Reviews on Google"
}
  • googleBusinessProfile - where the review link points
  • ratingValue - rating from 0.0 to 5.0 (use decimal format like 4.0)
  • reviewLinkText - the clickable label next to the stars

QR Code Modal

{
  "qrHeading": "Scan the QR Code",
  "qrDescription": "to view my Business Card on another device"
}

Sharing (Mobile Share Button)

These values are used by the “Share” button (native sharing on mobile devices). 

{
  "shareTitle": "Your Name Digital Business Card",
  "shareText": "You can view my Digital Business Card here:"
}

4. Update index.html (Important for Social Sharing)

Your vCard content is loaded from cardData.json, but social platforms do not run JavaScript. When someone shares your link on WhatsApp, Facebook, LinkedIn, Messenger, or X/Twitter, the preview card is generated only from the static meta tags inside vcard/index.html.

Update these in index.html for every new vCard:
  • Basic metadata: <title>, <meta name="author">, <meta name="description">
  • Open Graph (Facebook/LinkedIn/WhatsApp): og:title, og:description, og:image, og:url
  • Twitter/X: twitter:title, twitter:description, twitter:image, twitter:url

Important: og:image, twitter:image, og:url, and twitter:url should use absolute URLs to your live site (social crawlers don’t reliably resolve relative paths). The image URL must point to a publicly accessible file (no logins, no private folders). 

<!-- Open Graph (Facebook, LinkedIn) -->
<meta property="og:type" content="website" />
<meta property="og:title" content="Liam Thorley Digital Business Card" />
<meta property="og:description" content="The digital business card of Liam Thorley - ready to save, share, and connect." />
<meta property="og:image" content="https://vcard-demo.co.uk/test22/vcard/images/share-1200x630.png" />
<meta property="og:image:width" content="1200" />
<meta property="og:image:height" content="630" />
<meta property="og:url" content="https://vcard-demo.co.uk/test22/vcard/" />

<!-- Twitter / X -->
<meta name="twitter:card" content="summary_large_image" />
<meta name="twitter:title" content="Liam Thorley Digital Business Card" />
<meta name="twitter:description" content="The digital business card of Liam Thorley - ready to save, share, and connect." />
<meta name="twitter:image" content="https://vcard-demo.co.uk/test22/vcard/images/share-1200x630.png" />
<meta name="twitter:url" content="https://vcard-demo.co.uk/test22/vcard/" />
Preview not updating? Social platforms cache previews aggressively. Use these tools to force a refresh:
  • Facebook Sharing Debugger
  • LinkedIn Post Inspector

5. Preview Locally (Recommended)

To preview everything properly (QR, share modal, buttons, JSON loading), use a local web server. This template uses a strict Content Security Policy (CSP), so it will not behave correctly if you open index.html directly from your file system (file://).

  • Visual Studio Code + Live Server (recommended)
  • LocalWP if you prefer a beginner-friendly local environment
Expected warning if opened incorrectly:
If you run the template by double-clicking index.html (or from the wrong folder root), your browser may show console errors such as CSP blocks and CORS failures (for example: blocked images, blocked inline scripts, or “origin null” when loading cardData.json). This is normal in that scenario.

✅ These warnings will not appear once:
  • you run the project through a proper local server (e.g. Live Server), and
  • the vCard is uploaded and served from a real web host (same origin as the files).
Important: Do not open index.html by double-clicking it (file://). Always use a local server so the JSON file loads correctly under CSP.

VS Code tip: Open the project folder in VS Code (the folder that contains demo-1 and demo-2), then right-click demo-1/vcard/index.html and choose “Open with Live Server”.

6. Deploy Your vCard

  1. Edit vcard/cardData.json with your real details
  2. Replace images in vcard/images/ (keep dimensions similar)
  3. Update vcard/index.html meta tags + absolute URLs for sharing (Open Graph/Twitter/icons)
  4. (Optional) Update colours in vcard/styles/style.css
  5. Upload the entire vcard folder to your hosting
  6. Access it at: https://yourdomain.com/vcard/ (or /vcard/index.html)
Changes not showing up? It's usually caching.
Try a hard refresh (Ctrl+F5 / Cmd+Shift+R). If using Cloudflare, a caching plugin, or server caching, purge the cache there too.

7. Multi-User Setups (Optional)

If you want multiple vCards on one domain (team members / clients), give each person their own folder: 

/liam/vcard/
/jane/vcard/
/john/vcard/

Each folder should contain its own cardData.json and images, and each folder must have its own updated index.html meta tags (especially og:url and og:image). The NFC card for each person simply points to their unique URL.

8. Styling & CSS Customisation

All colours and spacing are controlled using CSS variables at the top of styles/style.css. Update the variables under :root to change the theme. 

:root {
  --color-primary: #0a40ab;
  --color-light-bg: #ffffff;
  --color-dark-text: #000000;
  --color-light-text: #ffffff;
}
If you want a more neutral look, you can override the social platform background colours in CSS.
