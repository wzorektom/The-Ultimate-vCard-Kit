How to Edit and Deploy Your Digital vCard

This guide explains how to customise your vCard using the provided vcard folder. Your vCard is JSON-driven, which means your contact content (name, phone, links, images, share text, etc.) is stored in one place: vcard/cardData.json.

Main workflow:
1) Edit cardData.json for content
2) Edit index.html only for social share preview + absolute URLs (Open Graph / Twitter / icons)

1. Folder Structure

The folder you received is named vcard. All paths are configured to work with this folder name. Do not rename it unless you also update all related references and URLs. 

vcard/
  index.html
  cardData.json
  vcard.bundle.js
  styles/style.css
  scripts/qrcode.min.js
  images/...

2. Files Overview

  • cardData.json – main file you edit (text, links, images, share settings)
  • index.html – page shell + social sharing meta tags (Open Graph / Twitter)
  • vcard.bundle.js – reads cardData.json and renders the page + vCard download
  • styles/style.css – styling + colour variables
  • scripts/qrcode.min.js – QR code library (do not edit)
  • images/ – profile image, cover image, favicon, share preview image(s)
Important: If you change image filenames, update the matching paths in cardData.json (and in index.html for social share images/icons). Keep the same image dimensions where possible.

3. Edit Your Content in cardData.json

Open vcard/cardData.json. This file contains your profile details and settings. You can safely edit values inside quotes. Avoid removing commas, brackets, or keys.

Common Fields

FieldWhat it controls
fullNameYour full name (page + vCard download)
jobTitleShown under your name
companyYour business / brand
birthdayBirth date in yyyymmdd format (example: 19900101)
emailEmail button + vCard
telNumberMain phone (call button + vCard)
officeNumberOptional office/landline
whatsappNumberUsed for WhatsApp button
websiteUrlWebsite link
addressAddress shown on page + vCard
bioShort personal intro
companyDescriptionLonger description inside the company section

Images

Replace your images inside vcard/images/, then ensure the paths in cardData.json match: 

{
  "profileImage": "images/headshoot-300x300.png",
  "coverImage": "images/cover-960x640.png"
}

Social Profiles

Social links are configured in the socialProfiles array. Each item supports: type, url, and enabled.

Show / hide a profile: set "enabled": true or false.
If enabled is missing, it’s treated as enabled by default. 
"socialProfiles": [
  { "type": "youtube", "url": "https://youtube.com/yourlogin", "enabled": true },
  { "type": "tiktok", "url": "https://tiktok.com/@yourlogin", "enabled": false },
  { "type": "linkedin", "url": "https://linkedin.com/in/yourlogin" }
]

Reviews (Star Rating + Link)

{
  "googleBusinessProfile": "https://g.page/yourbiz",
  "ratingValue": "4.5",
  "reviewLinkText": "Check Our Reviews on Google"
}
  • googleBusinessProfile – where the review link points
  • ratingValue – rating from 0.0 to 5.0 (use decimal format like 4.0)
  • reviewLinkText – the clickable label next to the stars

QR Code Modal

{
  "qrHeading": "Scan the QR Code",
  "qrDescription": "to view my Business Card on another device"
}

Footer

{
  "footerTagline": "Smart connections start with a digital card.",
  "footerCompanyUrl": "https://yourwebsite.com/",
  "footerCompanyName": "Your Company Ltd"
}

Sharing (Mobile Share Button)

These values are used by the “Share” button (native sharing on mobile devices). 

{
  "shareTitle": "Your Name Digital Business Card",
  "shareText": "You can view my Digital Business Card here:"
}

4. Update index.html (Important for Social Sharing)

Even though your vCard content is loaded from JSON, social platforms do not run JavaScript. That means your preview on WhatsApp, Facebook, LinkedIn, Messenger, and X/Twitter depends on the hardcoded meta tags inside vcard/index.html.

You MUST update these in index.html for each card:
  • <title> and the basic meta tags (author, description)
  • Favicon + Apple icon links (must be absolute URLs on your domain)
  • Open Graph tags (og:title, og:description, og:image, og:url)
  • Twitter/X tags (twitter:title, twitter:description, twitter:image, twitter:url)

These URLs must point to publicly accessible files on your live site. Example (replace with your domain and path): 

<meta property="og:image" content="https://yourdomain.com/vcard/images/share-1200x630.png">
<meta property="og:url" content="https://yourdomain.com/vcard/">

<meta name="twitter:image" content="https://yourdomain.com/vcard/images/share-1200x630.png">
<meta name="twitter:url" content="https://yourdomain.com/vcard/">

<link rel="icon" href="https://yourdomain.com/vcard/images/favi-100x100.png">
<link rel="apple-touch-icon" sizes="128x128" href="https://yourdomain.com/vcard/images/bookmark-128x128.png">
You can test share previews using:
  • Facebook Sharing Debugger
  • LinkedIn Post Inspector
If the preview doesn’t update, use these tools to refresh the cached preview.

5. Preview Locally (Recommended)

To preview everything properly (QR, share modal, buttons, JSON loading), use a local web server such as:

  • Live Server extension in Visual Studio Code
  • LocalWP if you prefer a beginner-friendly local environment
Important: Do not open index.html by double-clicking it (file://). Use a local server so the JSON file loads correctly.

6. Deploy Your vCard

  1. Edit vcard/cardData.json with your real details
  2. Replace images in vcard/images/ (keep dimensions similar)
  3. Update vcard/index.html meta tags + absolute URLs for sharing (Open Graph/Twitter/icons)
  4. (Optional) Update colours in vcard/styles/style.css
  5. Upload the entire vcard folder to your hosting
  6. Access it at: https://yourdomain.com/vcard/ (or /vcard/index.html)
Changes not showing up? It’s usually caching.
Try a hard refresh (Ctrl+F5 / Cmd+Shift+R). If using Cloudflare, a caching plugin, or server caching, purge the cache there too.

7. Multi-User Setups (Optional)

If you want multiple vCards on one domain (team members / clients), give each person their own folder: 

/liam/vcard/
/jane/vcard/
/john/vcard/

Each folder should contain its own cardData.json and images, and each folder must have its own updated index.html meta tags (especially og:url and og:image). The NFC card for each person simply points to their unique URL.

8. Styling & CSS Customisation

All colours and spacing are controlled using CSS variables at the top of styles/style.css. Update the variables under :root to change the theme. 

:root {
  --color-primary: #0a40ab;
  --color-light-bg: #ffffff;
  --color-dark-text: #000000;
  --color-light-text: #ffffff;
}
If you want a more neutral look, you can override the social platform background colours in CSS.
